# DianDian (点点) 视觉设计系统 & UI/UX 规范
**Design Language: "Nebula Path / 星途"**

> **设计哲学：** 极简主义、智能协同、绝对信任。  
> 我们不是在做一个测试工具，而是在打造一个能够听懂人类、辅助人类的“智能外挂”。设计必须消除“代码”带来的沉重感，转而提供“丝滑”的指挥感。

---

## 1. 核心视觉理念 (Core Values)

### 1.1 极简主义 (Minimalism)
*   **不显眼的 UI**: 界面应退居幕后，让用户专注于“测试目标”和“浏览器画面”。
*   **负空间**: 大胆使用空白和间距（Spacing），通过层级而非边框来区分功能区。

### 1.2 呼吸感与动态监控 (Liveness)
*   **AI 的“心跳”**: 当 AI 在思考（Thought）时，界面应有细微的呼吸灯或波纹效果，证明它正在“活着”并处理逻辑。
*   **实时反馈**: 每一个指令（Command）到浏览器动作（Action）的过程都应该是透明且带有物理惯性的动画。

---

## 2. 视觉资产 (Visual Identity)

### 2.1 配色方案 (Color Palette)
采用 **"深空霓虹 (Deep Space Neon)"** 主题，平衡生产力工具的严谨与 AI 时代的科技感。

| 类别 | 颜色 (HEX) | 用途 |
| :--- | :--- | :--- |
| **Space Black** | `#0A0A0B` | 主背景，极深灰色，避免纯黑带来的压抑感。 |
| **Surface Dark** | `#141415` | 卡片、侧边栏背景。 |
| **AI Purple** | `#8B5CF6` | AI 相关元素、主要操作按钮（类似 GPT 调色）。 |
| **Nebula Blue** | `#3B82F6` | 链接、进度条、次要高亮。 |
| **Succes Mint** | `#10B981` | 测试通过、成功状态。 |
| **Warning Ember** | `#F59E0B` | 需要用户介入、AI 卡顿状态。 |
| **Error Crimson** | `#EF4444` | 测试失败、API 报错。 |

### 2.2 字体规范 (Typography)
*   **标题/交互**: `Inter` 或 `Outfit` (现代、几何感强，视觉重心稳)。
*   **代码/日志**: `JetBrains Mono` (工程感，易读性极高)。

## 3. 组件交互语言 (Interaction Language)

### 3.1 玻璃拟态 (Glassmorphism)
*   对话气泡和浮动面板采用 `12px` 圆角，配合 `backdrop-filter: blur(8px)` 和 1px 的半透明边框（`border: 1px solid rgba(255,255,255,0.05)`）。

### 3.2 SoM (视觉标记) 设计
这是产品的灵魂。当 AI 标注网页元素时：
*   **标签样式**: 细线框（1px dashed）配合左上角极小的数字角标。
*   **高亮动画**: AI 选中的元素会有一个从 0.5 到 1.0 的淡入高亮，伴随轻微的向内缩放效果。

### 3.3 对话流 (The Flow)
*   **用户指令**: 右侧对齐，深紫色背景，简洁圆润。
*   **AI 响应**: 左侧对齐，卡片化展示。
    *   **思考折叠区 (Thought)**: 默认收起或精简显示，点击可展开完整的逻辑链路（Trace）。
    *   **动作项 (Action)**: 带有小图标（如 🖱️ Click, ⌨️ Type）的显著按钮。

## 4. 关键页面布局方案

### 4.1 首页 (Dashboard) - "The Launchpad"
*   **中心化输入框**: 类似 Raycast 的全宽输入框。初始态只显示一行：“今天想测点什么？”，输入 URL 后展开详细指令。
*   **历史 Bento Grid**: 采用不等高网格展示最近测试，失败的测试用红色模糊光晕包围。

### 4.2 运行页 (The Runner) - "The Cockpit"
*   **6:4 比例**: 6 份给浏览器视图（关键资产），4 份给侧边聊天栏。
*   **侧边栏固定底部**: 输入框固定，指令流向上滚动。
*   **全局控制条**: 浮动于浏览器画面正上方，包含：[⏸ 暂停] [⏹ 终止] [✋ 接管] [📸 手动截图]。
